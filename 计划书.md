
---
🚀 Alpha-Insight: 自主量化投研智能体集群 (2026)
Alpha-Insight是一款基于 LangGraph编排、E2B 沙箱执行、并集成 动态爬虫与 Telegram 实时推送的全栈自主量化投研 Agent。它不仅能阅读新闻，更能像数据科学家一样编写代码、执行回测、纠正错误，并主动向用户汇报行情异动。

🤖 2. 核心 Agent 角色定义
Agent 角色	核心逻辑模型	职责描述
首席分析师 (Planner)	DeepSeek-R1	利用长链推理 (CoT) 拆解需求，决定使用 API 还是动态爬虫。
数据采集员 (Scraper)	GPT-4o / V3	针对无 API 网页进行动态抓取，并将非结构化信息转为结构化数据。
量化开发者 (Coder)	Claude 3.5	生成 Python 代码进行 Pandas 处理、TA-Lib 计算及 Plotly 绘图。
错误审计员 (Debugger)	GPT-4o	监控沙箱报错，分析 Traceback，提供精准的代码修复建议。
合规质检员 (Reviewer)	DeepSeek-V3	过滤数值幻觉，确保分析结论符合金融常识，生成研报。

🛠️ 3. 技术栈 (Tech Stack 2026)
大脑 (LLMs): DeepSeek-R1 (推理), Claude 3.5 Sonnet (代码), GPT-4o (通用)
编排 (Orchestration): LangGraph (支持循环逻辑的状态机)
沙箱 (Sandbox): E2B (云端隔离 Python 运行环境)
数据 (Data): yfinance, Crawl4AI (动态爬虫), Tavily (金融搜索)
通讯 (Delivery): Telegram Bot API, Streamlit
观测 (Observability): Arize Phoenix / LangSmith

📅 4. 30天实战开发路线图
📅 第一周：感知层 - 数据采集与安全沙箱
Day 1: Docker 开发基座落地。提供 `Dockerfile` + `docker-compose.yml`（app 与测试服务），固定 Python 版本并使用 lock 文件安装依赖；采用分层构建 + 缓存卷（如 `/root/.cache/pip`）+ 预装镜像策略，确保容器每次启动不重复下载/构建依赖。

Day 2: 实现 `core/sandbox_manager.py` 异步沙箱管理器（E2B SDK），完成会话创建、执行、销毁；所有外部调用使用 `async/await`，节点入口统一输出 `[DEBUG] QuantNode {name} Start`。

Day 3: 封装 `tools/market_data.py`（yfinance 适配器），输出标准化 DataFrame；强制 UTC 时区与 `fillna()`；当数据为空时返回“数据未找到”。

Day 4: 集成 Crawl4AI，落地 `agents/scraper.py` 与清洗管线（HTML -> Markdown -> 结构化字段）；支持 API 缺失时的回退抓取。

Day 5: 在 LangGraph 中实现第一版路由：`Planner -> (API or Scraper)`，并用 Pydantic 定义 `AgentState` 最小闭环字段，禁止未定义状态流转。

Day 6: 接入 Telegram Bot 推送链路（文本 + 图片），打通 `tools/telegram.py`；输出沙箱生成图表并完成移动端通知。

Day 7: 完成第一周集成验证与质量门禁：运行测试脚本覆盖“请求 -> 规划 -> 数据获取 -> 沙箱执行 -> Telegram 推送”；验证 Python Traceback 已结构化供 Debugger 使用，并记录 Docker 冷启动/热启动耗时对比（证明依赖未重复下载）。

✅ 第一周交付（已落地到仓库）
- Docker 环境：`Dockerfile.dev` + `docker-compose.yml`（`dev` 与 `test` 服务），并启用 pip 缓存卷 `pip-cache`，默认复用镜像层与缓存，避免每次启动重复下载依赖。
- 沙箱执行：新增 `core/sandbox_manager.py`，E2B SDK 优先，缺省回退本地 Docker 沙箱；支持 `create/execute/destroy` 异步生命周期。
- 数据工具：新增 `tools/market_data.py`，yfinance 适配器输出标准化记录，强制 UTC 与 `fillna()`，空数据返回“数据未找到”。
- 动态抓取：新增 `agents/scraper.py`，Crawl4AI 优先，aiohttp 回退；新增 `agents/planner.py` 与 `agents/workflow.py`，实现 `Planner -> (API or Scraper) -> Sandbox` 的 LangGraph 路由闭环。
- 通讯能力：新增 `tools/telegram.py`，支持文本与图片推送。
- 质量门禁：新增 `tests/test_week1_workflow.py`、`tests/test_market_data.py` 和 `scripts/week1_integration.py`，并提供 `scripts/docker_startup_benchmark.sh` 记录 Docker 冷/热启动时间。

📅 第二周：认知层 - 多智能体编排与纠错循环
Day 8: 设计 LangGraph状态拓扑，定义 AgentState 数据结构。

Day 9: 编写 Planner Agent(DeepSeek-R1) 系统提示词，实现任务的多级拆解。

Day 10: 实现 Planner 的“工具路由”功能，使其能根据需求自主选择爬虫或 API。

Day 11: 构建 Coder Agent，建立沙箱代码生成标准（统一库导入、统一变量命名）。

Day 12: 核心攻关：实现 Self-Correction 闭环，Debugger 自动解析沙箱报错并反馈。

Day 13: 优化 Debugger 提示词，支持修复列名错误、库缺失、逻辑异常等常见 Bug。

Day 14: 接入 LangGraph Checkpointer，实现对话进度的持久化与断点续传。

Day 15: 中期测试：连续运行 10 个复杂量化任务，记录代码自修复成功率。

✅ 第二周交付（已落地到仓库）
- 状态拓扑：新增 `agents/week2_workflow.py`，实现 `Planner -> Coder -> Executor -> Debugger -> Coder` 的 LangGraph 自修复循环，支持重试上限控制。
- Planner 深度规划：新增 `agents/week2_planner.py`，内置 DeepSeek-R1 CoT 系统提示词与 JSON 输出约束；当未配置 `DEEPSEEK_API_KEY` 时自动回退到本地规划器。
- Coder 代码标准：新增 `agents/week2_coder.py`，统一沙箱代码模板（`pandas`、UTC 归一化、固定变量命名、仅通过 Python 执行产出数值）。
- Debugger 纠错：新增 `agents/week2_debugger.py`，基于结构化 Traceback 生成修复建议，覆盖列名错误、依赖缺失、语法错误与数据类型错误。
- Checkpointer：Week2 图默认接入 `InMemorySaver`，支持对话线程 `thread_id` 级别的状态持久化。
- 质量门禁：新增 `tests/test_week2_workflow.py`，覆盖任务拆解、故障注入后的自修复闭环、Checkpointer 写入验证。

📅 第三周：专业层 - 量化建模与多模态输出
Day 16: 在沙箱中集成 Plotly，实现交互式 HTML 报告生成。

Day 17: 编写图形提取脚本，将沙箱内的 .html 或 .png 转化为可传输的 Base64/文件流。

Day 18: 集成基础回测逻辑（基于 Pandas），支持 Agent 计算策略胜率与最大回撤。

Day 19: 数值门禁系统：强制要求 LLM 的所有数值结论必须引用自沙箱 Python 输出。

Day 20: 构建多模态研报模板，将文字分析、指标图表、回测数据整合为一体化消息。

Day 21: 实现 Human-in-the-Loop节点，高风险操作（如策略买入信号）需等待人工确认。

Day 22: 优化逻辑链：让 Planner 能在多轮对话中记忆之前的代码执行结果。

Day 23: 压力测试：模拟极端行情（如熔断），测试 Agent 的新闻情绪响应速度。

✅ 第三周交付（已落地到仓库）
- 交互式绘图与报告：新增 `agents/week3_coder.py`，在沙箱脚本中输出 Plotly HTML（失败时回退纯 HTML），并生成 PNG/PDF 报告文件。
- 前端可传输产物：扩展 `core/sandbox.py` / `core/sandbox_manager.py` 支持提取 `.html/.png/.pdf`；新增 `tools/artifact_extractor.py` 将产物编码为 Base64 负载。
- 技术指标与回测：在 Week3 代码模板中集成 MACD/RSI（TA-Lib 优先，pandas 回退）和向量化回测（策略收益、基准收益、胜率、最大回撤）。
- 数值门禁：新增 `agents/week3_reviewer.py`，强制从沙箱 `METRICS_JSON` 解析数值并生成报告，避免 LLM 直接“口算”产出结论。
- 多模态融合研报：`agents/week3_workflow.py` 将情绪分（Scraper 输入）与技术分融合为综合分并生成最终建议。
- HITL 中断：Week3 图新增 `hitl` 节点，`BUY` 建议时通过 LangGraph `interrupt` 等待人工确认后再继续。
- 记忆与持续上下文：Week3 状态维护 `metrics_history`，结合 checkpointer 支持多轮累计执行结果。
- 质量门禁：新增 `tests/test_week3_workflow.py`，覆盖指标回测脚本、产物提取、融合评分/HITL、以及 checkpointer 记忆能力。

📅 第四周：应用层 - 实时推送与交互驾驶舱

Day 25: 完善 Telegram 推送逻辑，支持不同优先级（异动预警 vs. 常规日报）。

Day 26: 搭建 Streamlit 驾驶舱，实时展示 Agent 的思考链 (Thought) 与代码流。

Day 27: 接入 Arize Phoenix，实现对 Agent 全链路追踪与 Token 消耗监控。

Day 28: 企业级安全加固：限制沙箱权限，禁止非法网络访问与本地文件读写。

Day 29: 性能调优：减少冗余 Token 传递，优化多 Agent 之间的信息同步效率。

Day 30: 项目 Landing：编写 README，录制演示视频，准备面试 Demo。

✅ 第四周交付（已落地到仓库）

💡 5. 面试精华策略 (Interview Prep)
关于算数幻觉：
    “我从不信任 LLM 的口算能力。在 Alpha-Insight 中，所有的数学逻辑都被转化为 Python 代码在 E2B 沙箱中运行。LLM 只负责‘写程序’，而不直接‘产出数字’。”

关于自修复能力：
    “通过 LangGraph 的循环图架构，我实现了一个 Debugger 节点。当 Coder 生成的代码报错时，Debugger 会分析回溯信息并指导 Coder 进行自我修正。这模拟了真实开发者的开发工作流。”

关于数据获取：
    “我的系统具备‘眼’的功能。除了标准的 yfinance API，我还集成了 Crawl4AI 动态爬虫 Agent，它能实时抓取那些没有 API 的金融媒体信息，实现了数据的全量化覆盖。”
