Codex + Ollama 本地子代理部署与使用文档
版本：2026-03-08（Ollama 原生接口版）
目标：让 Codex 按需调用本地 Ollama 子代理，减少大型项目遍历带来的主模型 token 消耗
推荐架构：Codex 负责决策与修改；本地 Ollama 模型负责搜索、定位、局部阅读和压缩摘要。
接入方式：使用 Codex 官方支持的 MCP，由 Codex 在 ~/.codex/config.toml 或 .codex/config.toml 中加载 stdio MCP 服务器。
省 token 的关键：MCP 工具只返回候选文件、局部片段和短摘要，不把完整遍历过程和大段文件全文回灌给 Codex。
1. 适用场景与总体架构
本文档适用于 Linux 远程开发环境。典型场景是：你通过 VSCode Remote-SSH 或 Codex CLI 在远程机器上工作，希望 Codex 在修改大型项目时不要自己遍历整个仓库，而是按需调用一个本地“代码探索子代理”。
本方案中，Codex 仍然是主代理，负责任务拆解、修改决策、补丁生成和收尾修复；本地模型不直接做最终修改，而是通过 MCP 工具层承担项目遍历、代码搜索、局部读取和摘要压缩。
角色	职责
Codex	主代理；决定什么时候调用 MCP 工具；综合证据后生成修改方案和补丁
本地 Ollama 模型	本地探索模型；在 MCP 服务器内部负责候选文件分析、局部摘要和变更影响说明
MCP 服务器	把 list/search/read/summarize 这类能力以标准 MCP tools 形式暴露给 Codex

2. 前置条件
•	远程 Linux 服务器 1 台，建议至少 16 GB 内存；有 GPU 更好，CPU 也可以先验证链路。
•	已安装 Codex CLI 或 Codex IDE Extension，并能在远程环境中访问项目目录。
•	Python 3.10+、git、curl、build-essential（或同类编译工具）、ripgrep。
•	已安装 Ollama，且本机可访问 127.0.0.1:11434。
•	网络可访问模型来源；如果你用的是本地 GGUF，也可以直接导入到 Ollama。
3. 使用 Ollama 部署本地模型
3.1 安装 Ollama
按官方方式安装即可。Linux 环境下建议优先按官方安装说明完成基础服务安装，再根据需要通过 systemd override 调整环境变量。
3.2 导入模型的两种方式
方式 A：如果该模型已经有可直接运行的 Ollama 名称
ollama pull your-model-name
ollama run your-model-name

方式 B：如果你手上是 GGUF 文件
Ollama 支持通过 Modelfile 导入 GGUF。Modelfile 中使用：
FROM /path/to/file.gguf

然后执行：
ollama create my-model

3.3 推荐做法
如果你原来准备用 LocoOperator-4B-GGUF，最自然的做法就是把 GGUF 放到固定目录，然后用 Modelfile 包装成可运行的 Ollama 模型。
mkdir -p ~/models/loco-operator
cd ~/models/loco-operator

创建 Modelfile：
FROM /home/youruser/models/loco-operator/locooperator-4b.Q4_K_M.gguf
PARAMETER temperature 0.2
PARAMETER num_ctx 16384

构建模型：
cd ~/models/loco-operator
ollama create loco-operator -f Modelfile

3.4 启动与健康检查
curl http://127.0.0.1:11434/api/version
curl http://127.0.0.1:11434/api/tags

再验证模型对话：
curl http://127.0.0.1:11434/api/chat \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "loco-operator",
    "stream": false,
    "messages": [
      {"role": "system", "content": "You are a concise codebase exploration assistant."},
      {"role": "user", "content": "Explain in one sentence what a search sub-agent should do."}
    ]
  }'

4. 创建 MCP 服务器
这里采用 MCP Python SDK 的 FastMCP，通过 stdio 方式直接运行。Codex 会拉起这个 Python 进程，并通过 MCP 调用其中暴露的工具。

4.3 完整 MCP 服务器代码（Ollama 原生接口版）
#!/usr/bin/env python3
import json
import os
import subprocess
from pathlib import Path
from typing import List

import requests
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("loco-explorer")

PROJECT_ROOT = Path(os.environ.get("PROJECT_ROOT", os.getcwd())).resolve()
OLLAMA_API_BASE = os.environ.get("OLLAMA_API_BASE", "http://127.0.0.1:11434")
OLLAMA_MODEL = os.environ.get("OLLAMA_MODEL", "loco-operator")

TEXT_EXTS = {
    ".py", ".ts", ".tsx", ".js", ".jsx", ".java", ".go", ".rs", ".cpp", ".c", ".h",
    ".hpp", ".cs", ".rb", ".php", ".swift", ".kt", ".kts", ".scala", ".sql", ".sh",
    ".yaml", ".yml", ".toml", ".json", ".md", ".txt", ".ini", ".cfg",
}


def _safe_path(path_str: str) -> Path:
    candidate = (PROJECT_ROOT / path_str).resolve() if not os.path.isabs(path_str) else Path(path_str).resolve()
    if not str(candidate).startswith(str(PROJECT_ROOT)):
        raise ValueError(f"Path escapes PROJECT_ROOT: {candidate}")
    return candidate


def _run(cmd: List[str], cwd: Path | None = None, timeout: int = 30) -> str:
    proc = subprocess.run(
        cmd,
        cwd=str(cwd or PROJECT_ROOT),
        capture_output=True,
        text=True,
        timeout=timeout,
        check=False,
    )
    if proc.returncode != 0:
        raise RuntimeError(proc.stderr.strip() or f"command failed: {' '.join(cmd)}")
    return proc.stdout


def _call_ollama_chat(system_prompt: str, user_prompt: str) -> str:
    url = f"{OLLAMA_API_BASE}/api/chat"
    payload = {
        "model": OLLAMA_MODEL,
        "stream": False,
        "messages": [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        "options": {"temperature": 0.1}
    }
    r = requests.post(url, json=payload, timeout=120)
    r.raise_for_status()
    data = r.json()
    return data["message"]["content"].strip()


@mcp.tool()
def list_project_files(subdir: str = ".", max_results: int = 200) -> str:
    base = _safe_path(subdir)
    out = _run(["rg", "--files", str(base)], timeout=30)
    lines = [x for x in out.splitlines() if x.strip()][:max_results]
    return json.dumps({
        "root": str(base),
        "count": len(lines),
        "files": lines,
        "truncated": len(out.splitlines()) > max_results,
    }, ensure_ascii=False, indent=2)


@mcp.tool()
def search_code(query: str, subdir: str = ".", glob: str = "", max_hits: int = 60) -> str:
    base = _safe_path(subdir)
    cmd = ["rg", "-n", "-S", "--hidden", "-g", "!.git/**"]
    if glob:
        cmd.extend(["-g", glob])
    cmd.extend([query, str(base)])
    out = _run(cmd, timeout=40)
    hits = []
    for line in out.splitlines()[:max_hits]:
        parts = line.split(":", 2)
        if len(parts) == 3:
            try:
                line_no = int(parts[1])
            except ValueError:
                continue
            hits.append({"file": parts[0], "line": line_no, "text": parts[2][:240]})
    return json.dumps({
        "query": query,
        "hits": hits,
        "returned": len(hits),
        "truncated": len(out.splitlines()) > max_hits,
    }, ensure_ascii=False, indent=2)


@mcp.tool()
def read_file_excerpt(path: str, start_line: int = 1, end_line: int = 120) -> str:
    file_path = _safe_path(path)
    if not file_path.is_file():
        raise ValueError(f"Not a file: {file_path}")
    if file_path.suffix not in TEXT_EXTS:
        raise ValueError(f"Unsupported text extension: {file_path.suffix}")
    start_line = max(1, start_line)
    end_line = max(start_line, min(end_line, start_line + 400))
    lines = file_path.read_text(encoding="utf-8", errors="ignore").splitlines()
    excerpt = []
    for idx in range(start_line - 1, min(end_line, len(lines))):
        excerpt.append(f"{idx + 1}: {lines[idx]}")
    return json.dumps({
        "path": str(file_path),
        "start_line": start_line,
        "end_line": end_line,
        "content": "\n".join(excerpt),
    }, ensure_ascii=False, indent=2)


@mcp.tool()
def summarize_for_change(task: str, file_paths: str, max_lines_per_file: int = 120) -> str:
    paths = [p.strip() for p in file_paths.split(",") if p.strip()]
    if not paths:
        raise ValueError("file_paths is empty")
    chunks = []
    for raw in paths[:8]:
        fp = _safe_path(raw)
        if not fp.is_file():
            continue
        text = fp.read_text(encoding="utf-8", errors="ignore").splitlines()[:max_lines_per_file]
        numbered = "\n".join(f"{i+1}: {line}" for i, line in enumerate(text))
        chunks.append(f"### FILE: {fp}\n{numbered}")
    if not chunks:
        raise ValueError("No readable files found")
    system_prompt = (
        "You are a concise read-only codebase exploration assistant. "
        "Given a change task and a small set of candidate files, explain which files matter, "
        "why they matter, and what Codex should inspect next. Keep the answer compact and structured. "
        "Return valid JSON."
    )
    user_prompt = (
        f"Task:\n{task}\n\n"
        "Candidate files (truncated):\n\n"
        + "\n\n".join(chunks)
        + "\n\nReturn JSON with keys: likely_targets, why, risks, next_reads."
    )
    return _call_ollama_chat(system_prompt, user_prompt)


def main() -> None:
    mcp.run()


if __name__ == "__main__":
    main()

5. 配置 Codex 使用 MCP
Codex 官方支持在 ~/.codex/config.toml 或项目内 .codex/config.toml 配置 MCP 服务器。对于 stdio MCP server，可配置 command、args、env、cwd 等字段。
5.1 项目级 .codex/config.toml 示例
model = "gpt-5-codex"
approval_policy = "on-request"
sandbox_mode = "workspace-write"

[sandbox_workspace_write]
network_access = true

[mcp_servers.loco_explorer]
command = "/home/youruser/loco-mcp/.venv/bin/python"
args = ["/home/youruser/loco-mcp/loco_mcp_server.py"]
cwd = "/home/youruser/work/your-repo"

[mcp_servers.loco_explorer.env]
PROJECT_ROOT = "/home/youruser/work/your-repo"
OLLAMA_API_BASE = "http://127.0.0.1:11434"
OLLAMA_MODEL = "loco-operator"

5.2 用户级 ~/.codex/config.toml 也可以
如果你希望所有项目共享同一个 MCP 子代理，可以写到 ~/.codex/config.toml；但对多项目环境，更建议把 PROJECT_ROOT 放在项目级 .codex/config.toml 中。
5.3 验证 Codex 已加载 MCP
codex mcp --help

# 在 Codex TUI 中可使用：
/mcp

6. 具体使用流程
步骤	操作说明
步骤 1：确认 Ollama 正常工作	先执行 curl http://127.0.0.1:11434/api/version 和 /api/tags，必要时先用 ollama run loco-operator 确保模型能启动。
步骤 2：进入项目目录	确保 .codex/config.toml 可被当前项目加载，并且 PROJECT_ROOT 指向当前仓库。
步骤 3：启动 Codex	CLI 和 IDE Extension 都会读取同一套 Codex 配置层；项目级配置只会在项目被信任时生效。
步骤 4：让 Codex 调用子代理	用自然语言提出修改目标，例如先调用本地代码探索工具列出最相关文件、调用链入口和配置来源。
步骤 5：让 Codex 开始修改	在子代理返回候选文件和摘要后，Codex 再决定读取哪些片段、如何改动和如何验证。

7. 推荐提示词
•	提示词 1：探索优先：先不要修改代码。先调用本地代码探索工具，列出最相关的文件、调用链入口和配置来源，再给我一个最小改动方案。
•	提示词 2：控制 token：目标是减少主模型遍历仓库的 token 消耗。除非必要，不要读取完整大文件；优先使用 search_code、read_file_excerpt 和 summarize_for_change。
•	提示词 3：先 shortlist：请先通过子代理判断哪些文件最值得读，再进入实际修改。输出时把候选文件控制在 3 到 8 个。
8. 省 token 的设计原则
•	不要把目录树、grep 全量输出和大文件全文直接回给 Codex。
•	先 search，再 read excerpt，最后 summarize。
•	把 summarize_for_change 的输入限制在 3 到 8 个候选文件，每个文件最多 80 到 120 行。
•	当任务进入真正修改阶段后，再让 Codex 读取更长片段；不要在探索阶段提前喂太多代码。
9. 常见问题排查
问题	处理方法
Codex 没看到 MCP 工具	检查 config.toml 路径、项目是否已被信任、Python 虚拟环境中的 mcp 是否安装，以及 command/args/cwd 是否写对。
MCP 启动超时	先手动运行 /home/youruser/loco-mcp/.venv/bin/python /home/youruser/loco-mcp/loco_mcp_server.py，确认没有 import error 或路径问题。
summarize_for_change 报连接错误	先检查 curl http://127.0.0.1:11434/api/version 和 /api/tags 是否可达；如果这里不通，优先修 Ollama。
搜索结果太多	把 max_hits 调小，或优先限定 subdir / glob。
没有明显省 token	通常是因为工具返回太长；优先缩短摘要和片段，而不是增加更多工具。

10. 官方依据与说明
•	OpenAI Codex MCP 文档：Codex 使用 config.toml 配置 MCP，支持 stdio server，CLI 与 IDE Extension 共享配置。
•	OpenAI Codex 配置参考：mcp_servers.<id>.command、args、env、cwd 等字段可用。

•	Ollama API 版本说明：Ollama API 不严格做版本号管理，但目标是稳定和向后兼容，因此本文档统一采用原生 /api 接口。
•	Ollama Modelfile / Import 文档：支持通过 FROM /path/to/file.gguf 导入 GGUF 并用 ollama create 构建模型。
•	MCP Python SDK 官方文档：FastMCP 可直接通过 @mcp.tool() 暴露工具。
11. 最终建议
这套方案最适合大型项目修改前的探索阶段。真正节省 Codex token 的，不是“多了一个本地模型”本身，而是你把遍历、grep、初筛和摘要压缩留在本地，只把最少必要信息交给 Codex。
在这版改造中：Codex 继续作为主代理；MCP 继续作为工具层；Ollama 取代 llama.cpp 成为本地推理后端；接口统一改成 Ollama 原生 /api/chat，不再依赖 /v1 兼容层。
