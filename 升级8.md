# Alpha-Insight Upgrade8：产品体验与24x7分析升级（Telegram优先）
**基线截至：2026-02-28**

> 目标：在 Upgrade7 工程底座上，完成“用户可感知”的产品层升级。  
> 原则：先让用户在首屏看懂“涨跌结论”，再给原因、证据与动作；降低刷屏与操作负担，提升对话自然度与可信度。

---

## 0. 背景与定位

Upgrade7 解决了“后端可维护、可治理、可扩展”的底层问题。  
现阶段主要瓶颈已从“系统能不能跑”转为“用户第一眼是否得到答案、是否愿意持续使用”。

Upgrade8 聚焦 Telegram 产品体验，形成“结论先行 + 可验证证据 + 低噪声交互 + 24x7持续价值”的闭环。

---

## 1. 升级目标（2026 Q2）

### 1.1 产品目标
- 首屏直接回答用户问题（先给涨跌，再给原因）。
- 降低交互噪声（进度消息单条更新，结果单次发出）。
- 新闻信息从“数量展示”升级到“主题与影响”。
- 歧义标的从“阻断等待”升级为“可理解、可快速确认”的选择器。
- 首屏按钮收敛，减少选择困难。

### 1.2 体验目标
- 同一请求最多两条核心消息：`进度消息` + `最终结果卡片`。
- 默认不展示内部运行字段（run_id/request_id），仅在调试模式可见。
- 用户在 3 秒内看到“已受理/处理中”，在结果消息中直接看到涨跌结论。

### 1.3 运营目标
- 引入 24x7 运行状态可见化（`/status` 已具备，升级8完善展示口径）。
- 形成“主动价值触达”能力：在监控链路外补充市场脉冲摘要（可灰度）。

---

## 2. 非目标（防止范围失控）

- 不在 Upgrade8 内重写 Upgrade7 的底层架构（GraphState/Policy/Connector 不重做）。
- 不引入自动交易执行能力（依旧坚持 research/alert 范畴）。
- 不做全渠道一致改造，优先 Telegram 完整闭环，再复用到其他渠道。
- 不在本阶段改造策略引擎核心逻辑，重点在输出组织与交互节奏。

---

## 3. 当前问题诊断（面向用户体感）

### 3.1 首屏回答不足
- 用户问“一个月涨跌”，系统首屏常先给技术模板而非涨跌结论。
- 涨跌额、区间位置、振幅等高价值字段未标准化呈现。

### 3.2 歧义选择体验生硬
- 命中多个候选时阻断流程，且选择策略不透明。

### 3.3 消息噪声高
- 同 request 可能出现多条阶段性与重复分析消息，造成“不稳定”感。

### 3.4 新闻价值密度低
- “近7天X条”偏日志化，不足以支持快速判断。

### 3.5 证据对用户不友好
- 证据字段偏内部口径，缺少“来源、更新时间、指标口径”这类可验证信息。

---

## 4. Upgrade8 信息架构（First Screen Contract）

## 4.1 首屏必须字段（优先级顺序）
1. `近30天涨跌幅(%) + 涨跌额(价格差)`
2. `区间高/低 + 振幅 + 最大回撤`
3. `当前价在区间位置（高位/中位/低位）`
4. `技术一句话：方向 + 触发条件`
5. `新闻一句话：主题Top3 + 时间窗 + 来源覆盖`
6. `下一步动作按钮`

## 4.2 技术一句话标准
- 结构：`方向判断 + 触发条件 + 风险说明`
- 示例：
  - `短线偏弱，若站回10日均线并放量可转中性；否则维持震荡偏弱。`

## 4.3 新闻模块标准
- 不再只报条数，改为：
  - 主题 Top3（财报/监管/产品/宏观/其他）
  - 每主题一句影响摘要
  - 每主题1条代表新闻（标题+时间+来源）

## 4.4 首屏按钮标准
- 首屏仅保留：`📈K线` `📰新闻` `🔔提醒` `更多`
- 其余按钮（报告/重试/近3个月/只看新闻）收进“更多”。

---

## 5. 交互协议升级（Telegram Conversation Contract v3）

## 5.1 消息节奏
- 每个请求只允许：
  - 1条进度消息（编辑更新）
  - 1条最终结果卡片
- 禁止同 request_id 发送重复最终卡片。

## 5.2 去噪规则
- 默认隐藏：`run_id/request_id/schema/action/internal metric keys`。
- 内部证据进入 audit/store/evidence，不进主消息。

## 5.3 歧义选择器
- 候选展示标准：`代码 + 市场 + 公司名`（最多5条）。
- 支持自然语言确认：`选港股那个` / `选美股`。
- 默认选择策略可配置：
  - 中文语境默认 HK
  - 或按用户历史偏好优先

---

## 6. 24x7 分析能力增强（在现有调度基础上增量）

## 6.1 运行状态卡（Status Card）
- 统一口径展示：活跃监控、24h告警、投递成功、重试深度、降级状态。
- `/status` 作为用户可观测入口，保持简洁与可解释。

## 6.2 主动脉冲摘要（可灰度）
- 新增 `市场脉冲` 任务（建议每1h或4h）：
  - Top movers（涨跌幅）
  - 新闻主题变化
  - 风险提示
- 仅在用户开启订阅后主动推送，默认关闭。

---

## 7. 输出模板（可直接落地）

```text
9988.HK 阿里巴巴-SW｜近30天（YYYY-MM-DD ~ YYYY-MM-DD）

涨跌：-X.XX%（-Y.Y）
区间：低 L / 高 H（振幅 A.A%）｜最大回撤 -D.D%
现价：C（区间位置：低位/中位/高位）

技术：RSI14=28.5，短线偏弱；若站回10日均线并放量，信号可转中性。
新闻：近7天 N 条，主题Top3：①财报 ②监管 ③业务进展。

证据：行情源=XXX 更新时间=YYYY-MM-DD HH:mm；新闻源覆盖=...；指标口径=RSI14(1d)。

按钮：📈K线｜📰新闻｜🔔提醒｜更多
```

## 8. 路线图（P0 / P1 / P2）

### P0（止血，必须）
1. 首屏补齐涨跌/区间/位置字段，并按“结论优先”排序。
2. 同 request_id 结果去重；进度消息改“编辑同一条”。
3. 收敛首屏按钮到 4 个主入口。
4. 证据块改为用户可验证三件套（来源、更新时间、口径）。

### P1（核心体验升级）
1. 歧义选择器升级（候选标准化 + 自然语言确认 + 默认策略透明）。
2. 新闻主题化输出（Top3主题 + 每主题一句影响 + 代表新闻）。
3. 技术一句话改“方向+触发条件”。

### P2（运营增强）
1. 市场脉冲摘要（订阅制主动推送）。
2. Status Card 增强（历史趋势、异常提示、恢复状态）。
3. 多渠道复用 Telegram 输出契约。

## 11. 风险与回滚策略

### 11.1 风险
- 首屏字段增加导致消息超长。
- 进度编辑与并发消息冲突。
- 歧义默认策略误选带来理解偏差。

### 11.2 回滚
- 通过 feature flag 回退至 Upgrade7 输出模板。
- 关闭进度编辑，回退为静态单条“处理中”。
- 关闭市场脉冲主动推送，仅保留手动查询。

---

## 12. 完成定义（Definition of Done）

- 用户问“一个月涨跌”，首屏必含涨跌结论与区间位置。
- 每请求最多两条核心消息（进度1 + 结果1）。
- 默认不暴露内部字段（run/request/schema/action）。
- 新闻区可读性提升：主题化 + 影响摘要 + 来源可追溯。
- `/status` 与能力卡可一致呈现“7x24运行能力”。

---

## 13. 与 Upgrade7 的关系

- Upgrade7 是“工程底座升级”。
- Upgrade8 是“产品可感知升级”。

两者关系：
- Upgrade7 提供可治理与可观测能力，保证 Upgrade8 能稳定落地。
- Upgrade8 把 Upgrade7 的能力转化为用户看得懂、愿意持续使用的体验价值。
